Complex Number Calculator

I worked in a group of five people using the Agile developmental process to create this complex number calculator over the span of six weeks. We followed a product owner’s specifications and presented our iterations every two weeks. The calculator can do regular math, complex number math, has a history pane, and has a complex number calculator that shows the left operand, right operand, and the evaluation of the equation.

Spreadsheet

This was made using Ruby and is a small sized spreadsheet that accepts a variety of types such as Integers, Floats, Strings, Cell References, and Booleans. The spreadsheet uses Ruby Curses for the TUI and has 3 windows, one for the spreadsheet itself, one for entering formulas, and one for showing a cell’s value or an error message since the cell itself may be too small to show the full message/value. For the formula window, the user is able to assign values to variables, use the variables, create conditional if statements, and create for-each loops.

Memory Game

This is another game I made over the summer to review Java. It has a 3x3 grid, but could easily be changed to fit a 4x4, or even larger grid to play on. The user enters their name and the game begins. A single square in the grid is highlighted briefly and the user clicks it. Then two light up in a specific order and the user has to remember the order and match it when they click the squares. Each level adds one more box to remember and this continues until you reach the max level of 100 (which can easily be changed to any number) or until you misclick the order. (Github)

TicTacToe Game

This was made over the summer after my sophomore year to review Java and practice my coding skills. It is a simple Tic-Tac-Toe game where two players’ names are entered and starts with the first player clicking in the grid to place an “X”. It then switches to the second player’s turn and they place an “O”. The game continues until someone gets 3 X’s or O’x in a row, or the board is full. At the end of the game, the users are given the option to quit or restart. (Github)

Java Hybrid Deque

Doubly-linked-list implementation of the java.util.Deque interface. This implementation is more space-efficient than Java's LinkedList class for large collections because each node contains a block of elements instead of only one. This reduces the overhead required for next and previous node references.

Breaking NTRU

This program is given a relatively small sized lattice of 46x46 and then reduces the lattice down using the Lenstra–Lenstra–Lovász lattice basis reduction algorithm (LLL). With the reduced lattice, I get each of the vectors in the lattice which are just the rows, and I also get the negated vector of each row as well. I then find the shortest vectors and use each of them to try to make a private key that can decrypt a chosen message encrypted by the public key.

Hangman

I thought it would be fun to try making a simple game of hangman, so I created this game in Java. The first player enters a hidden phrase that the second player has to guess. They also enter the amount of lives that the guesser should get. Once those two things are put in, the game begins and the guesser is able to enter letters that they think fit in the hidden phrase.
